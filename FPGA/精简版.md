## 第一部分：核心理论知识点

### 1. 设计方法与流程
- **自顶向下设计**：先定义系统的整体功能和顶层架构，然后将系统逐步拆分为更小的子模块，直至具体的门级电路实现。这种方法的优点在于适合复杂系统开发，便于早期进行仿真验证和错误定位。
- **自底向上设计**：先从基础的门电路、触发器、计数器等基本元件开始搭建，然后将其组合成更大的功能模块。这种方法效率较低，且容易出错，不适合大规模复杂设计。
- **IP核复用**：IP核是指功能完整、经过验证且可重复使用的电路功能模块。根据其实现形式，分为软IP（以硬件描述语言代码形式提供）、固IP（已完成综合）和硬IP（最终物理掩膜版图）。在不同设计中复用这些已验证的IP核，可以显著提高开发效率。
- **标准设计流程**：首先进行设计输入（原理图或HDL代码），然后通过综合工具将高层描述转化为门级网表，接着进行适配（布局布线）以生成针对目标芯片的编程文件（如Bitstream），最后进行仿真验证（包括功能仿真和时序仿真）并将配置文件下载到器件中。

### 2. CPLD与FPGA器件的结构与区别
- **可编程逻辑阵列基础**：早期的可编程逻辑器件（PLD）内部主要由与阵列和或阵列构成。
  - **PROM**：与阵列固定（全译码），或阵列可编程。
  - **PLA**：与阵列可编程，或阵列也可编程。
  - **PAL**：与阵列可编程，但或阵列固定。
  - **GAL**：与阵列可编程，或阵列固定，但其输出电路增加了可组态的宏单元（OLMC），可以灵活配置为组合输出或时序输出。
- **CPLD（复杂可编程逻辑器件）**：基于**乘积项（与或阵列）**结构，采用EEPROM或Flash工艺制造。因此它具有**非易失性**，掉电后数据不会丢失，上电即可工作。其内部通过全局互连总线连接多个逻辑块。它的优点是传输延时固定且可预测，适合实现复杂的组合逻辑和状态机，但逻辑容量相对较小。
- **FPGA（现场可编程门阵列）**：基于**查找表（LUT）**结构，查找表本质上是一个小容量的静态随机存取存储器（SRAM），预先存储了所有输入组合对应的输出值。大多数FPGA采用SRAM工艺，因此它是**易失性**的，掉电后数据会全部丢失，每次上电都需要从外部配置芯片（如Flash）读取配置文件。FPGA内部包含大量可配置逻辑块（含LUT和触发器）、块RAM和DSP单元。它的优点是逻辑容量极大，并行处理能力极强，适合复杂算法和高速接口设计，但其延时与具体布线路径相关，无法像CPLD那样完全预测。
- **主要工艺与编程元件**：
  - **SRAM**：绝大多数FPGA使用，可无限次编程，但掉电丢失。
  - **EEPROM/Flash**：多数CPLD使用，非易失，可反复擦写。
  - **反熔丝**：一次性编程，具有抗辐射特性，常用于航天或军工领域。
- **JTAG边界扫描（IEEE 1149.1标准）**：该接口包含 **TDI**（测试数据输入）、**TDO**（测试数据输出）、**TMS**（测试模式选择）、**TCK**（测试时钟）和可选的 **TRST**（复位）。它主要有三大用途：检测PCB板级焊接故障、通过ISP（在系统编程）功能配置器件、以及配合逻辑分析仪内核（如SignalTap）进行在线调试。

### 3. Verilog HDL核心语法与要素
- **常量表示**：整数常量的标准格式为 `位宽'进制 数值`，例如 `8'hd5`。需要注意的是，位宽和进制符号（`'h`、`'b`等）之间**不允许**有空格。逻辑值包括 `0`（低电平）、`1`（高电平）、`x`（不定态）和 `z`（高阻态）。
- **数据类型**：
  - **wire型**（连线型）：对应物理连线。它的值随输入变化而实时变化，没有存储功能。必须使用 `assign` 连续赋值语句或通过模块实例化端口来驱动。
  - **reg型**（变量型）：可以在 `always` 或 `initial` 过程块中被赋值，具有保持值的特性。**需要特别注意**，`reg` 型变量在综合时不一定对应寄存器，如果描述的是组合逻辑（完整敏感列表且无时钟触发），综合器会将其映射为纯导线或组合逻辑，只有在时序逻辑中才会被映射为触发器。
- **常用运算符要点**：
  - **逻辑运算符**（`&&`、`||`、`!`）的结果是布尔值（真/假），操作数被视为整体（非零即为真）。
  - **按位运算符**（`&`、`|`、`^`）则是将操作数对应的每一位单独进行运算。
  - **相等运算符**：`==` 和 `!=` 在比较时，如果操作数中含有 `x` 或 `z`，则结果是不确定的（`x`）；而 `===` 和 `!==` 会逐位精确比较包括 `x` 和 `z` 在内的所有位，结果一定是确定值（0或1）。
  - **缩位运算符**（单目运算符）如 `&`、`^`，它们会将向量（多位宽信号）的所有位进行归约运算，最终结果只有1位。例如 `^4'b1010` 的结果是 `1^0^1^0 = 0`，常用于生成奇偶校验位。
  - **位拼接运算符** `{}`：将多个信号按位组合成一个新的向量，例如 `{cout, sum}` 将进位和求和结果拼接起来。

### 4. 行为语句与赋值方式（重点）
- **过程块**：`initial` 块在仿真开始时只执行一次，且不可综合，仅用于测试激励的编写。`always` 块则会根据敏感列表的条件反复执行，用于描述可综合的组合逻辑或时序逻辑。
- **敏感列表规则**：
  - 描述组合逻辑时，建议使用 `always @(*)`，这样综合器和仿真器会自动将所有输入信号加入敏感列表，避免因遗漏信号而导致仿真与综合行为不一致，或意外生成锁存器。
  - 描述时序逻辑时，敏感列表必须包含时钟边沿，如 `always @(posedge clk)`（同步复位）或 `always @(posedge clk or negedge rst_n)`（异步复位）。
- **阻塞赋值（`=`）与非阻塞赋值（`<=`）的本质区别**：
  - **阻塞赋值**：在 `begin...end` 串行块中按书写顺序逐条执行。执行某条阻塞赋值语句时，会立即计算右边的表达式并将结果赋给左边的变量，下一条语句会立即看到该变量更新后的值。因此，**书写顺序直接影响逻辑结果**。阻塞赋值通常用于描述组合逻辑。
  - **非阻塞赋值**：在一个 `always` 过程块开始执行时，先同时计算所有非阻塞赋值语句右边的表达式（此时读取的都是变量进入该过程块时的旧值），并暂存起来。当整个 `always` 块中的所有语句都执行完毕后，再将这些暂存的值同时赋给左边的变量。因此，**在同一时钟沿触发的多个非阻塞赋值是并行完成的，书写顺序不影响最终结果**。非阻塞赋值通常用于描述时序逻辑（如寄存器、计数器）。
- **关键设计准则**：
  1. 描述组合逻辑（`always @(*)`）时，使用阻塞赋值 `=`。
  2. 描述时序逻辑（`always @(posedge clk)`）时，使用非阻塞赋值 `<=`。
  3. 禁止在同一个 `always` 块中混合使用阻塞赋值和非阻塞赋值。
  4. 禁止在两个或多个不同的 `always` 块中对同一个变量赋值，这会产生多驱动冲突。

### 5. 有限状态机（FSM）设计
- **摩尔（Moore）型与米利（Mealy）型**：
  - Moore型状态机的输出**仅取决于当前状态**，与输入信号无关。它的输出在时钟边沿后的状态稳定时随之稳定，不易产生毛刺。
  - Mealy型状态机的输出**同时取决于当前状态和当前输入**。输入信号的变化会立即通过组合逻辑影响输出，因此响应速度更快，但输出信号上可能会出现毛刺。
- **状态编码**：
  - **顺序编码**：使用二进制自然数表示状态，需要的触发器数量最少，但状态转换时可能有多个位同时翻转，容易产生竞争冒险。
  - **格雷编码**：相邻状态之间只有1位不同，能有效降低功耗和减少瞬态毛刺。
  - **一位热码（One-Hot）**：每个状态使用单独的一位寄存器，N个状态需要N个触发器。在FPGA设计中这种编码方式最为常用，因为FPGA拥有丰富的触发器资源，一位热码可以极大简化译码逻辑，提高电路的最高工作频率。
- **标准三段式写法**：
  - **第一段**：时序逻辑，负责在时钟边沿将 `next_state` 赋值给当前状态寄存器 `state`，并处理复位。
  - **第二段**：组合逻辑（使用阻塞赋值 `=`），根据当前状态 `state` 和输入信号计算下一状态 `next_state`。该组合逻辑块中的 `case` 语句必须包含 `default` 分支，以确保所有未定义的状态（无效状态）都能跳转回起始状态，防止状态机“死锁”。
  - **第三段**：组合逻辑（使用阻塞赋值 `=`），根据当前状态（Moore型）或当前状态和输入（Mealy型）计算输出信号。同样需要确保所有分支都有明确的赋值，以避免生成锁存器。
- **定义状态常量**：推荐使用 `localparam` 在模块内部定义状态编码，它的作用范围仅限于本模块且不能被上层模块重新定义，比 `parameter` 更安全。

### 6. 设计优化与可靠性
- **流水线技术**：将原本在一个时钟周期内完成的长组合逻辑路径（关键路径）切割为多个较短的逻辑段，并在每一段的末尾插入触发器寄存中间结果。这样做可以显著缩短每级的组合逻辑延时，从而**提高系统最高时钟频率**。缺点是增加了触发器的使用数量（面积开销），并且数据从输入到输出会产生几个时钟周期的延迟（Latency），但数据的吞吐率（每个周期产出一个结果）保持不变。
- **资源共享**：通过增加少量多路选择器（MUX），让多个数据流分时共用同一个消耗面积较大的功能模块（如乘法器、加法器）。这是一种**用速度换面积**的策略，能有效减少硬件资源消耗，但会在数据路径上增加MUX的传输延时。
- **毛刺消除**：组合逻辑电路中，当多个输入信号同时变化时，由于各路径延时不同，输出端在稳定之前可能会产生尖峰脉冲（即毛刺）。最可靠且常用的消除方法是在该组合逻辑的输出端增加一级由系统时钟触发的D触发器，将组合逻辑的输出寄存一拍后再输出。由于触发器仅在时钟上升沿采样数据，而毛刺的宽度通常很窄，不满足触发器的建立时间要求，因此毛刺不会被采到，从而实现了干净输出。

### 1. 时钟分频模块（84MHz → 1KHz / 10Hz）
**基本原理**：利用一个计数器对输入时钟的周期进行计数。当计数值达到预设的翻转阈值时，将输出信号的电平翻转，从而产生频率更低的方法时钟。

- 分频系数 `N` = 输入时钟频率 / 输出时钟频率。
- 50%占空比方波的翻转阈值 = `N / 2`。例如，84MHz分频到1KHz，N=84000，阈值=42000。

```verilog
module clk_divider #(
    parameter DIV_CNT = 42000  // 用于1KHz: 42000；用于10Hz: 4200000
)(
    input               clk,    // 84MHz 输入
    input               rst_n,  // 低电平有效复位
    output  reg         clk_out // 分频后的方波输出
);

    reg [31:0] cnt;

    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) begin
            cnt     <= 32'd0;
            clk_out <= 1'b0;
        end else begin
            if (cnt == DIV_CNT - 1) begin  // 计数值达到阈值
                cnt     <= 32'd0;
                clk_out <= ~clk_out;       // 输出电平翻转
            end else begin
                cnt <= cnt + 1'b1;
            end
        end
    end
endmodule

// ---------- 顶层例化示例 ----------
module top (input clk_84m, input rst_n, output clk_1k, output clk_10hz);
    // 1KHz 分频器例化
    clk_divider #(.DIV_CNT(42000)) u_div1k (
        .clk(clk_84m), 
        .rst_n(rst_n), 
        .clk_out(clk_1k)
    );
    // 10Hz 分频器例化
    clk_divider #(.DIV_CNT(4200000)) u_div10hz (
        .clk(clk_84m), 
        .rst_n(rst_n), 
        .clk_out(clk_10hz)
    );
endmodule
```

### 2. 8位全加器（含进位）
**功能**：将两个8位输入数 `a` 和 `b` 以及一个进位输入 `cin` 相加，输出8位和 `sum` 以及最高位产生的进位 `cout`。推荐使用最简洁的数据流（`assign`）写法。

```verilog
module adder8 (
    input   [7:0]   a,
    input   [7:0]   b,
    input           cin,
    output  [7:0]   sum,
    output          cout
);
    // 数据流描述：拼接符号实现进位输出与求和
    assign {cout, sum} = a + b + cin;
endmodule
```

### 3. 七段数码管译码器（显示0~9，共阴极）
**功能**：输入4位BCD码（对应十进制0~9），输出7位段码以驱动共阴极数码管（高电平点亮）。段码排列顺序约定为 `{g, f, e, d, c, b, a}`。

```verilog
module seg_decoder (
    input   [3:0]   bcd,        // 输入十进制数 0~9
    output  reg [6:0] seg       // 七段码输出
);

    always @(*) begin
        case (bcd)
            4'd0: seg = 7'b011_1111; // 0x3F
            4'd1: seg = 7'b000_0110; // 0x06
            4'd2: seg = 7'b101_1011; // 0x5B
            4'd3: seg = 7'b100_1111; // 0x4F
            4'd4: seg = 7'b110_0110; // 0x66
            4'd5: seg = 7'b110_1101; // 0x6D
            4'd6: seg = 7'b111_1101; // 0x7D
            4'd7: seg = 7'b000_0111; // 0x07
            4'd8: seg = 7'b111_1111; // 0x7F
            4'd9: seg = 7'b110_1111; // 0x6F
            default: seg = 7'b000_0000; // 防止综合出锁存器
        endcase
    end
endmodule
```
*注：如果题目要求共阳极（低电平点亮），只需将所有输出段码按位取反即可。*

### 4. 序列检测器（检测输入“1111”，支持重叠检测）
**功能**：输入为1位串行数据 `x`，输出为1位检测标志 `z`。当连续检测到4个1（即“1111”）时，`z` 输出高电平。该设计支持重叠检测，例如输入“11111”时，`z` 会在检测到第4个1和第5个1时各输出一个周期的高电平。

**状态定义**：`IDLE`（未收到有效1）、`ONE`（已收到1个1）、`TWO`（已连续收到2个1）、`THREE`（已连续收到3个1）、`FOUR`（已连续收到4个1）。

```verilog
module seq_detector_1111 (
    input       clk,
    input       rst_n,      // 低电平有效异步复位
    input       x,          // 单比特串行输入
    output reg  z           // 检测标志输出
);

    // 使用 localparam 定义状态编码（采用一位热码）
    localparam IDLE = 5'b00001, ONE = 5'b00010, TWO = 5'b00100, THREE = 5'b01000, FOUR = 5'b10000;
    reg [4:0] state, next_state;

    // 第一段：时序逻辑，更新状态寄存器（异步复位）
    always @(posedge clk or negedge rst_n) begin
        if (!rst_n) state <= IDLE;
        else        state <= next_state;
    end

    // 第二段：组合逻辑，计算下一状态（使用阻塞赋值 =，必须包含 default）
    always @(*) begin
        case (state)
            IDLE:   next_state = (x == 1'b1) ? ONE  : IDLE;
            ONE:    next_state = (x == 1'b1) ? TWO  : IDLE;
            TWO:    next_state = (x == 1'b1) ? THREE: IDLE;
            THREE:  next_state = (x == 1'b1) ? FOUR : IDLE;
            FOUR:   next_state = (x == 1'b1) ? FOUR : IDLE; // 重叠检测：保持FOUR
            default:next_state = IDLE;  // 覆盖无效状态，防止死锁
        endcase
    end

    // 第三段：组合逻辑，计算输出（Moore型，只取决于当前状态）
    always @(*) begin
        if (state == FOUR) z = 1'b1;
        else               z = 1'b0;
    end

endmodule
```
*注：若题目要求“非重叠检测”（即检测到“1111”后立即清零重新开始），只需将 `FOUR` 状态下的次态 `next_state` 修改为 `IDLE`。*
